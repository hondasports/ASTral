# Stuck Advisor

## 自動トリガー

- 同じtestまたはcommandが2回以上、同じ理由で失敗した。
- 同じreview findingで2回以上差し戻された。
- OXC、SQLite、watcher、MCPの同じ前提を変えずに修正を繰り返している。

## 手順

1. 推測を分離して状況を整理する。

```text
目的:
正確な症状・エラー:
試したことと結果:
最後に成功した状態:
未確認事項:
```

2. 独立した仮説を3つ作る。1つは問題定義そのものを疑う。
3. 各仮説へ根拠、最小検証、棄却条件を書く。
4. 最も安全で速い検証を1つだけ実行する。
5. 外れた仮説を同じ形で繰り返さない。

## Grill Me連携

仮説が技術調査だけで決められず、要件、互換性、許容精度、運用上の優先順位へ依存する場合は`grill-me`へ移る。

- 一度に1問だけ聞く。
- 推奨回答を添える。
- コードやログから分かることはユーザーへ聞かない。
- `GRILL RESULT`を仮説選択の前提として記録する。

## 出力

```text
状況整理:
仮説A:
仮説B:
仮説C（問題定義の見直し）:
選んだ次の1アクション:
GRILL RESULT（該当時）:
停止・escalate条件:
```

3仮説を棄却しても進展がない場合は、追加情報と未確認範囲を示してescalateする。
